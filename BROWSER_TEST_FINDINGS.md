# 浏览器测试发现 - 问题与优化建议

## 测试环境
- 时间：2026-01-23
- 浏览器：Chrome
- 物料计划：1月5日-2月13日.xlsx (2026-01-05 至 2026-02-13)
- 物料数量：178个物料，25个多供应商物料

---

## 发现的问题

### 问题1: 物料列表加载进度提示 ⚠️ 中等优先级
**位置**：ShareAllocation.tsx - 物料份额分配页面
**现象**：
- 加载时显示"加载所有物料中... (0/178)"
- 有进度数字但没有进度条可视化
- 没有加载骨架屏预览

**影响**：
- 用户不知道加载进度具体进行到哪一步
- 等待体验不够友好

**优化建议**：
- [ ] 添加进度条（Progress Bar）显示加载进度
- [ ] 添加加载骨架屏（Skeleton Screen）预览表格结构
- [ ] 改进加载提示文案

**代码位置**：
- client/src/pages/ShareAllocation.tsx - 加载状态处理
- client/src/components/VirtualMaterialList.tsx - 列表渲染

---

### 问题2: 表格列宽和内容溢出 ⚠️ 中等优先级
**位置**：VirtualMaterialList.tsx - 多供应商物料表格
**现象**：
- "供应商分配"列内容较多，显示不完整
- 表格列宽度可能需要调整
- 在小屏幕上可能有溢出问题

**影响**：
- 用户需要滚动才能看到完整的供应商信息
- 用户体验不够顺畅

**优化建议**：
- [ ] 优化"供应商分配"列的显示方式
  - [ ] 使用省略号(...) + Tooltip显示完整信息
  - [ ] 或使用可展开的行详情
- [ ] 调整表格响应式布局
- [ ] 在移动设备上优化显示

**代码位置**：
- client/src/components/VirtualMaterialList.tsx - 列定义和渲染

---

### 问题3: 虚拟滚动性能 ⚠️ 低优先级
**位置**：VirtualMaterialList.tsx - 虚拟滚动实现
**现象**：
- 表格使用虚拟滚动（从元素索引看有很多"编辑份额"按钮）
- 滚动时可能有性能问题

**影响**：
- 大数据量滚动时可能卡顿
- 用户体验不够流畅

**优化建议**：
- [ ] 检查虚拟滚动的行高配置
- [ ] 优化行渲染性能
- [ ] 添加滚动性能监控

**代码位置**：
- client/src/components/VirtualMaterialList.tsx - 虚拟滚动配置

---

### 问题4: "编辑份额"按钮反馈不足 ⚠️ 低优先级
**位置**：ShareAllocationDialog.tsx - 编辑对话框
**现象**：
- 点击"编辑份额"按钮后需要等待对话框打开
- 没有明显的加载反馈

**影响**：
- 用户不知道系统是否在响应
- 可能导致重复点击

**优化建议**：
- [ ] 添加按钮加载状态（disabled + spinner）
- [ ] 添加对话框打开动画
- [ ] 改进用户反馈

**代码位置**：
- client/src/components/VirtualMaterialList.tsx - 按钮点击处理
- client/src/components/ShareAllocationDialog.tsx - 对话框打开

### 问题5: 邮件列表页面过长 ⚠️ 高优先级
**位置**：GenerateEmail.tsx - 邮件生成页面
**现象**：
- 邮件列表显示在一个长页面上（4111像素以下）
- 没有分页或虚拟滚动
- 需要不断向下滚动才能看到所有邮件
- 按钮太多，造成视觉混乱

**影响**：
- 用户体验差
- 页面加载缓慢
- 难以找到特定的邮件

**优化建议**：
- [ ] 添加分页功能
- [ ] 或使用虚拟滚动显示邮件列表
- [ ] 简化按钮布局（使用更紧凑的操作菜单）
- [ ] 添加搜索功能
- [ ] 添加邮件生成进度条

**代码位置**：
- client/src/pages/GenerateEmail.tsx - 邮件列表渲染
- server/routers.ts - generateEmails API

---
