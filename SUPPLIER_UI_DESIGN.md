### 功能模块详解

#### 2.1 顶部栏
```
左侧: 平台Logo + "供应商协同平台"
中间: 供应商名称 · 供应商编号
右侧: 
  - 📧 消息通知（小红点显示未读数）
  - 👤 账号信息下拉
  - 退出登录
```

#### 2.2 统计卡片 (3列)
| 卡片 | 内容 | 颜色 |
|------|------|------|
| 待确认物料 | 数量 + 较昨日变化 | 橙色/琥珀 |
| 即将交货 | 数量 + 最早日期 | 蓝色 |
| 本月完成率 | 百分比 + 趋势 | 绿色 |

#### 2.3 物料清单（核心功能）

**生产状态汇报（核心创新点）**:
- 每个交货日期**独立汇报进度**
- 6个状态按钮横向排列，点击即切换
- 当前状态高亮蓝色，已完成的状态显示绿色
- 系统自动推导物料整体进度（取最高状态）

**确认操作**:
- ✅ 确认交期 - 接受当前计划
- ✏️ 修改交期 - 可编辑日期和数量，添加/删除批次
- ❌ 无法交付 - 标记异常

**状态标识**:
- ✅ 已交足 → 绿色
- ️ 有缺数 → 橙色
- 已完成 → 灰色淡化

#### 2.4 确认记录
- 列表展示历史确认
- 每条记录显示: 计划名称、状态、确认时间

## 3. 管理员端 vs 供应商端 对比

| 维度 | 管理员端 | 供应商端 |
|------|----------|----------|
| 登录地址 | `/login` | `/supplier-login` |

## 5. 技术实现要点

### 路由隔离
```
管理员: /login → / (Dashboard)
供应商: /supplier-login → /supplier-portal
```

### 认证隔离
- 管理员: JWT token, `auth` router
- 供应商: 独立 session, `supplierAuth` router

### UI 组件差异
- 供应商端使用独立的 Layout 组件
- 顶部栏简化，不显示管理员功能入口
- 物料清单使用折叠表格（SupplierConfirm.tsx 已实现）
