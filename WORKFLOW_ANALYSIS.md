## 二、核心业务流程

### 流程1: 物料计划导入流程
```
用户上传Excel
    ↓
解析Excel文件 (excelParser.ts)
    ↓
验证数据格式
    ↓
保存到material_plans表
    ↓
保存物料项到material_items表
    ↓
保存日期计划到daily_schedules表
    ↓
显示成功提示
```

**涉及文件**:
- client/src/pages/Home.tsx - 上传页面
- server/routers.ts - uploadMaterialPlan API
- server/excelParser.ts - Excel解析
- drizzle/schema.ts - 数据库表

---

### 流程2: 供应商管理流程
```
查看供应商列表
    ↓
添加新供应商 (可选)
    ↓
编辑供应商信息
    ↓
保存到suppliers表
    ↓
关联到物料计划
```

**涉及文件**:
- client/src/pages/SupplierManagement.tsx - 供应商管理页面
- server/routers.ts - supplier相关API
- drizzle/schema.ts - suppliers表

---

### 流程3: 物料份额分配流程
```
选择物料计划
    ↓
查询多供应商物料列表
    ↓
点击编辑份额
    ↓
修改供应商份额百分比
    ↓
保存到material_supplier_mappings表
    ↓
显示成功提示
```

**涉及文件**:
- client/src/pages/ShareAllocation.tsx - 份额分配页面
- client/src/components/VirtualMaterialList.tsx - 虚拟滚动列表
- client/src/components/ShareAllocationDialog.tsx - 编辑对话框
- server/db.ts - getMaterialsWithSuppliersByPlan
- server/routers.ts - upsertMaterialSupplierMapping API

---

### 流程4: 邮件生成与发送流程
```
选择物料计划
    ↓
生成邮件内容 (generateSupplierEmail)
    ↓
生成Excel附件 (generateSupplierPlanExcel)
    ↓
保存邮件记录到generated_emails表
    ↓
显示邮件预览
    ↓
用户确认发送
    ↓
发送邮件 (emailService)
    ↓
更新发送记录
    ↓
显示发送结果
```

**涉及文件**:
- client/src/pages/GenerateEmail.tsx - 邮件生成页面
- server/routers.ts - generateEmails / sendEmail API
- server/emailGenerator.ts - 邮件内容生成
- server/excelGenerator.ts - Excel生成
- server/emailService.ts - 邮件发送

---

## 四、已知问题发现

### 问题1: 大数据表格加载性能 ⚠️
**位置**: ShareAllocation.tsx - 物料份额分配页面
**现象**: 
- 当物料数量较多时，表格加载缓慢
- 虚拟滚动已实现，但可能需要进一步优化

**影响范围**: 
- 物料份额分配页面的响应速度
- 用户体验

**优化方向**:
- 检查虚拟滚动配置
- 优化行渲染性能
- 添加加载骨架屏

---

### 问题2: 邮件生成性能 ⚠️
**位置**: GenerateEmail.tsx / routers.ts - generateEmails API
**现象**:
- 生成大量供应商的邮件时可能较慢
- 需要并发处理优化

**影响范围**:
- 邮件生成页面的响应速度
- 用户等待时间

**优化方向**:
- 使用Promise.all并发处理
- 添加进度条显示
- 流式处理大数据

---

### 问题3: Excel文件生成性能 ⚠️
**位置**: excelGenerator.ts
**现象**:
- 大数据量Excel生成可能较慢
- 内存占用可能较高

**影响范围**:
- 邮件发送时的Excel生成
- 文件下载速度

**优化方向**:
- 流式写入Excel
- 分块处理数据
- 优化单元格格式化

---

### 问题4: 加载状态提示不足 ⚠️
**位置**: 全局 - 所有异步操作
**现象**:
- 某些操作缺少加载提示
- 用户不知道系统是否在处理

**影响范围**:
- 用户体验
- 用户信心

**优化方向**:
- 添加加载骨架屏
- 添加进度条
- 添加操作反馈提示
